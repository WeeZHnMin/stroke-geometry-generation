# 数据集设计笔记

对，真正往下走，数据集反而是最关键的一环。
而且你这个题目有个很大的优势：

它非常适合程序化生成数据。

这比很多视觉任务舒服得多，因为你可以同时控制：

- 几何真值
- 语言描述
- 结构约束
- 难度分级
- 多样性

所以关键不是“能不能生成”，而是：

怎么生成得高质量、多样化、还能真正锻炼模型能力。

我建议你把数据集目标拆成 4 层，不要一开始就想着“随机生成很多图”。

## 一、先明确：你到底要让模型学什么能力？

这一步最重要。
数据不是越花越好，而是要对准能力。

我会把目标能力分成这几类：

### 1. 局部几何连续性

模型要学会：

- 直线怎么续
- 转角怎么转
- 圆弧怎么平滑
- 断点后怎么继续

这对应基础笔感。

### 2. 全局闭合与结构完成

模型要学会：

- 多边形闭合
- 房间轮廓闭合
- 曲线最终回收
- 不要越画越漂

这对应全局形状意识。

### 3. 语言到空间约束

模型要学会：

- 左右上下
- 大小关系
- 数量关系
- 相对位置
- “靠近”“远离”“相邻”“包含”

这对应 prompt 理解。

### 4. 多对象与组合结构

模型要学会：

- 先后画多个图形
- 图形间不混淆
- 邻接关系
- 布局关系
- 层次结构

这对应复杂场景能力。

所以你生成数据时，不能只追求“形状多”，而是要确保每一类能力都被专门覆盖。

## 二、数据集应该分层设计，不要一锅炖

我非常建议你做成课程式数据集。

### Level 1：单图元基础集

先生成最基础的：

- 线段
- 折线
- 三角形
- 矩形
- 正多边形
- 圆
- 椭圆
- 简单弧线

作用：

- 学局部轨迹规律
- 学闭合
- 学基本 prompt 对齐

prompt 也简单：

- “draw a triangle”
- “draw a small circle”
- “draw a rectangle on the left”

### Level 2：带变形和噪声的单对象集

然后加入多样化：

- 拉伸三角形
- 非规则四边形
- 圆形轻微扰动
- 房间角略不规则
- 起点随机
- 顺时针/逆时针随机
- 采样密度随机
- 局部缺口/断点

作用：

- 防止模型只背模板
- 学稳健几何表征

### Level 3：多对象布局集

开始组合多个对象：

- 一个三角形 + 一个圆
- 左边矩形，右边小圆
- 上方两个点状对象，下方一个大轮廓
- 相交 / 不相交 / 相邻 / 包含

作用：

- 学多对象顺序
- 学对象间空间关系
- 学 pen_state 何时切换对象

### Level 4：结构化任务集

更贴你真正想做的：

- 房间轮廓
- 简单 CAD 草图
- 邻接图转轮廓
- 家具布局边框
- 多房间平面图
- 门洞/缺口/连接边

这时候才能开始体现研究价值。

## 三、什么叫“高质量”？

你刚才说得很对，程序生成不难，难的是高质量。

我觉得高质量至少有 5 个标准。

### 1. 几何合法

不能只是随机点拼一拼。
要保证：

- 闭合图形真闭合
- 房间轮廓不自交
- 邻接关系成立
- 圆弧是真平滑的
- 多边形顺序一致

否则模型会学脏数据。

### 2. 语言描述准确

prompt 不能乱写。
比如如果图形在右边，就别写“left”。
如果有两个圆，就别写“a circle”。

程序化生成的一个大优势是：
语言描述可以由真值反推
这样文本质量反而比人工标注更稳定。

### 3. 多样性是真多样，不是假随机

很多合成数据看起来变化很大，其实都在一个窄分布里。

真正要控制的多样性包括：

- 形状类型
- 尺度
- 长宽比
- 朝向
- 起笔点
- 点采样密度
- 顺/逆时针
- 缺口位置
- 对象数量
- 对象间关系

### 4. 难度有梯度

不能全是简单图，也不能一开始全上复杂布局。
你最好人为控制：

- 容易样本
- 中等样本
- 困难样本

这样训练更稳，评估也更清楚。

### 5. 输出形式统一

这一点特别重要。
比如你得统一好：

- 坐标归一化范围
- 采样方向规则
- 是否闭合时重复起点
- pen_state 规则
- 多 shape 的顺序规则

否则模型会多学很多无意义方差。

## 四、如何做“多能力”覆盖？

我建议你对每条样本都打上能力标签。
哪怕训练时不用，分析也非常有用。

比如：

- requires_closure
- requires_corner_turning
- requires_curvature
- requires_left_right_relation
- requires_counting
- requires_multi_shape
- requires_adjacency_reasoning

这样你后面评估时可以看：

- 模型在闭合任务上怎么样
- 在转角任务上怎么样
- 在关系理解上怎么样

这会让论文味道强很多。

## 五、断点数据怎么设计才有意义？

你刚才提到“带有断点的坐标数据”，这个非常值得做。
但断点不能只是随便删几个点。

可以分成几种更有意义的断点：

### 1. 抬笔断点

多个 shape 之间正常换笔。

### 2. 结构缺失断点

比如房间一段边缺了，让模型学“整体结构仍成立”。

### 3. 遮挡式断点

比如圆弧中间断了一段，仍然要继续出平滑走势。

### 4. 条件断点

prompt 指示：

- “an open rectangle”
- “a room outline with a door gap”

这种断点不是噪声，而是任务本身的一部分。

这会让模型不仅学“连续”，还学“有意义的不连续”。

## 六、语言怎么生成才能高质量？

我建议不要只用模板一句话，而是做多层语言生成。

### 模板层

最基础的：

- “draw a triangle”
- “draw a circle on the right”

### 属性层

加入：

- 大小
- 位置
- 数量
- 朝向
- 形状风格

比如：

- “draw a small wide rectangle near the top”

### 关系层

加入多对象关系：

- left of
- above
- intersecting
- adjacent to
- inside

### 结构层

加入任务描述：

- “draw a room with an opening on the right wall”
- “draw two adjacent rooms sharing one wall”

这样模型的语言能力才会真正被调动起来。

## 七、我最建议的数据生成方式

我会推荐你用一个 scene graph -> geometry -> stroke sequence -> text 的流水线。

也就是：

### Step 1. 先采样场景图

决定：

- 有几个对象
- 对象类型
- 对象关系
- 全局布局约束

### Step 2. 再生成几何真值

把每个对象实例化成：

- 点集
- 轮廓
- 房间角点
- 圆弧采样点

### Step 3. 再转成笔迹序列

决定：

- 起点
- 顺/逆时针
- 采样密度
- 断点
- pen_state

### Step 4. 再由真值反推文本

把场景属性转成 prompt。

这样做好处很大：

- 几何一定是合法的
- 文本一定对真值
- 结构关系不会乱
- 评测标签天然齐全

这是我最推荐的方式。

## 八、如果想要研究价值更强，最好再做一个 benchmark 拆分

比如：

### Split A：简单 seen shapes

基础形状，训练和测试分布接近。

### Split B：组合泛化

训练看过单图形，测试看图形组合。

### Split C：语言泛化

测试时 prompt 换表达方式，但几何任务相同。

### Split D：结构泛化

测试更复杂的邻接图或更多角点房间。

这样你就能测：

- 几何泛化
- 组合泛化
- 语言泛化
- 结构泛化

这会比只报一个平均 loss 强很多。

## 九、现在最重要的是数据

现在比较重要的是数据了，如何生成大量的高质量的数据，能让模型收敛稳定，产生泛化能力的数据，这才是重中之重。

这里其实有 3 个目标，不是一个：

- 让模型收敛
- 让模型稳定
- 让模型泛化

这三件事对应的数据要求不完全一样。

收敛靠的是分布干净、表示统一、难度可控。
稳定靠的是几何合法、序列规则一致、噪声不过量。
泛化靠的是组合多样性、语言多样性、结构层级和分布拆分。

所以如果要认真做，我建议不要再停留在“多造点图形”，而是把数据生成系统直接设计成一个可控的数据工厂。

### 核心原则

不要直接随机生成 stroke。
要走这条链：

`scene spec -> geometry -> stroke sequence -> text -> metadata`

也就是：

1. 先采样一个场景规范
2. 再生成合法几何
3. 再转成笔迹序列
4. 再由真值反推文本
5. 再附上能力标签和难度标签

这样生成的数据才会干净、可控、可扩展。

### 如果目标是“能收敛”，最重要的是这 6 件事

#### 1. 坐标系统一

所有图形都放在统一画布，比如 `[0,1] x [0,1]`。

而且要统一：

- 起点规则
- 是否闭合时重复起点
- 顺时针/逆时针策略
- 每步 `dx,dy` 的尺度范围

如果这些规则乱，模型前期很难收敛。

#### 2. 序列规则统一

比如明确：

- 到 shape 起点：`move`
- shape 内部：`draw`
- shape 结束：`end_shape`
- 样本结束：`end_all`

这类规则一旦固定，模型的状态学习会稳定很多。

#### 3. 难度分层

不要一开始就混进太多复杂样本。
建议训练数据按 curriculum 出：

- `easy`
  单图元、规则图形、短序列
- `medium`
  变形图元、随机起点、采样密度变化
- `hard`
  多对象、关系约束、房间轮廓、缺口、邻接

这会明显改善收敛稳定性。

#### 4. 图形合法性检查

每个样本生成后必须过几何校验。
至少要检查：

- 是否自交
- bbox 是否越界
- 是否闭合正确
- shape 之间关系是否符合设定
- move 段是否真的不落笔

程序化生成的最大坑就是：
你以为你生成的是好数据，其实混进去很多坏样本。

#### 5. 分布不要太极端

如果一开始既有超短序列，又有超长复杂房间轮廓，模型会很难学。
建议控制：

- 每阶段序列长度范围
- 每类 shape 占比
- 多对象样本比例
- room / CAD 类样本比例

也就是要做采样配比。

#### 6. Prompt 先从简单稳定开始

语言也会影响收敛。
如果一开始 prompt 表达过于花哨，模型会把问题复杂化。

最好分阶段：

- 第一阶段：固定模板
- 第二阶段：加入属性词
- 第三阶段：加入关系句式
- 第四阶段：加入表达改写

### 如果目标是“稳定”，最重要的是降低无意义方差

这里很多人会误以为多样性越大越好，其实前期不是。

稳定训练更依赖：
减少不必要的随机性

比如先决定这些东西哪些固定，哪些随机。

#### 起步阶段建议固定或弱随机的

- 画布范围固定
- 采样方向先固定一种
- 采样密度先在小范围波动
- 图形大小不要跨度过大
- prompt 模板先不要过多同义改写

#### 后期再放开的

- 顺/逆时针
- 随机起点
- 描述改写
- 更大尺度变化
- 更复杂图形组合

也就是说：

先让模型学规则，再让模型学鲁棒性

### 如果目标是“泛化”，数据设计就不能只是随机扰动

泛化真正来自这几类变化。

#### 1. 组合泛化

训练只见过：

- 单三角形
- 单圆
- 单矩形

测试时给：

- 三角形 + 圆
- 两个矩形 + 一个小圆

这样能测模型是不是学会了组合，而不是死记模板。

#### 2. 属性泛化

训练时大多是：

- 小圆
- 大矩形

测试时给：

- 瘦高矩形
- 扁椭圆
- 细长三角形

这样看它是不是学了“形状规律”，不是只背平均形状。

#### 3. 关系泛化

训练见过：

- left of
- above

测试给：

- inside
- adjacent to
- surrounding

或者同关系不同表达：

- “to the left of”
- “on the left side of”

#### 4. 结构泛化

训练只见过简单房间，测试给：

- L 型房间
- 带开口房间
- 双房间共享墙
- 更高角点数轮廓

#### 5. 语言泛化

训练用固定模板，测试用改写 prompt。
这样才能知道模型是不是学了“条件-几何映射”，而不是死记句子。

### 真正高质量的数据工厂应该怎么设计

我会建议把数据生成器分成 5 个模块。

#### 1. `Spec Sampler`

负责采样任务规格。

输出像这样：

- 图形数量
- 图形类型
- 关系约束
- 尺度范围
- 是否有缺口
- 难度等级
- 语言模板等级

这一步不直接出点。

#### 2. `Geometry Builder`

根据 spec 生成几何真值。

比如：

- triangle -> 3 顶点
- circle -> sampled contour
- room -> polygon with constraints

这里要保证几何合法。

#### 3. `Stroke Compiler`

把几何真值编译成 stroke 序列。

决定：

- 从哪起笔
- 是否随机起点
- 顺/逆时针
- 采样点密度
- `move/draw/end_shape/end_all`

这一步是模型训练直接吃的目标序列。

#### 4. `Text Realizer`

把 spec 和 geometry 渲染成 prompt。

分层控制：

- 简单模板
- 属性模板
- 关系模板
- 改写模板

#### 5. `Metadata Annotator`

自动打标签：

- requires_closure
- requires_curvature
- requires_multi_shape
- relation_type
- difficulty
- num_shapes
- sequence_length

这个模块对后续分析和 benchmark 特别值钱。

### 想要“大量”但不“脏”，靠的不是一次生成很多，而是参数空间设计好

最好控制这些自由度：

- `shape_type`
- `size`
- `aspect_ratio`
- `rotation`
- `position`
- `num_points`
- `start_point_strategy`
- `direction_strategy`
- `relation_type`
- `gap_type`
- `language_template_id`

只要这些维度设计好，程序就能稳定地产出很多不重复但仍然合法的样本。

也就是说，数据量来自：

结构化参数组合

而不是：

完全随机乱采样

### 数据生成工作按三个阶段推进

#### 阶段 1：收敛优先

目标：
让模型真的学会画。

数据：

- 单对象
- 短序列
- 规则图形
- 简单 prompt
- 少量双对象

重点：

- 数据干净
- 规则一致
- loss 稳定下降

#### 阶段 2：稳定优先

目标：
让模型不容易漂、不容易乱切换状态。

数据：

- 起点随机
- 采样密度变化
- 稍有变形
- `move` 更自然
- 简单开口/缺口

重点：

- 长一点序列仍稳定
- 闭合更准
- pen_state 更准

#### 阶段 3：泛化优先

目标：
让模型真正有通用二维平面生成能力。

数据：

- 多对象组合
- 关系多样
- 房间/CAD 轮廓
- 语言改写
- 结构泛化 split

重点：

- 新组合能不能画
- 新表达能不能理解
- 新结构能不能迁移

### 一个特别实际的建议：先做“采样配方表”

这是最容易被忽略、但最有用的东西。

比如先人工定一张表：

- 40% 单图元规则形状
- 25% 变形单图元
- 20% 双对象关系
- 10% room 类
- 5% gap/open contour 类

再在每一类里规定：

- 尺度范围
- 长度范围
- prompt 复杂度
- 是否允许旋转
- 是否允许缺口

这样数据工厂就开始“可控”了。

### 一句最关键建议

先别追求海量，先追求“每一条样本都知道自己在训练模型哪种能力”。

一旦这件事做到了，你再把规模放大就很自然。
否则哪怕生成一百万条，也可能只是大规模噪声。

### 最值得马上做的事

如果给优先级排序，现在立刻做这三件：

1. 把数据生成器升级成 `spec -> geometry -> stroke -> text` 的流水线
2. 给每条样本自动打能力标签和难度标签
3. 做一个采样配方表，明确每类样本比例

这三件做完，数据这件事就真的进入正轨了。
